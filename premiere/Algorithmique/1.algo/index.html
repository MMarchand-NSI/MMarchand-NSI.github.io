    Algorithmique

    Qu'est-ce qu'un algorithme ?

    Un algorithme, c'est une recette de cuisine pour un ordinateur : une suite d'instructions précises, qui s'exécutent dans l'ordre, et qui finissent toujours par s'arrêter.

    Pour qu'une recette soit valable, elle doit :

    • Décrire des étapes claires et exécutables
    • Se terminer en un nombre fini d'étapes
    • Produire un résultat correct

    Un algorithme peut s'écrire en français, en pseudocode ou en Python. Ce qui compte, c'est qu'il soit suffisamment précis pour ne laisser aucune ambiguïté.

    Exercice 1 - Est-ce un algorithme valable ?

    Parmi les descriptions suivantes, lesquelles constituent un algorithme valable ?

    1. "Ajouter du sel jusqu'à ce que ce soit bon."
    2. "Pour i allant de 0 à 9 : afficher i."
    3. "Tant que tu n'as pas trouvé, cherche encore."
    4. "Diviser le nombre par 2 jusqu'à obtenir 0, compter le nombre de divisions."
    Correction
    1. Non. "Jusqu'à ce que ce soit bon" n'est pas précis : la condition d'arrêt est subjective et ambiguë.
    2. Oui. Les étapes sont claires et la boucle se termine après exactement 10 itérations.
    3. Non. Si la valeur cherchée est absente, la boucle ne se termine jamais.
    4. Oui. La condition d'arrêt est claire (obtenir 0) et est forcément atteinte car diviser par 2 fait diminuer le nombre.

    Problèmes classiques

    Certains problèmes sont si courants en informatique qu'ils ont un nom officiel :

    Nom Description
    SORTING Trier une liste d'éléments
    SEARCH Trouver un élément dans une liste
    FIND-MIN Trouver le plus petit élément d'un tableau
    TSP Trouver le plus court chemin passant par toutes les villes
    PRIME Déterminer si un entier est premier

    Une grande partie de la recherche en informatique consiste à trouver des algorithmes de plus en plus rapides pour résoudre ces problèmes.

    Exercice 2 - Associer problème et description

    Associer chaque situation concrète au nom de problème correspondant dans le tableau ci-dessus.

    1. Google Maps cherche le chemin le plus court entre deux villes.
    2. On cherche si le mot "chat" est présent dans un dictionnaire.
    3. On range des élèves par ordre alphabétique.
    4. On détermine si 97 est un nombre premier.
    Correction
    1. TSP (ou PATH selon la variante)
    2. SEARCH
    3. SORTING
    4. PRIME

    Un premier exemple : trouver le minimum

    Problème FIND-MIN : étant donné un tableau de nombres, trouver le plus petit.

    Par exemple, pour le tableau [4, 9, 3, 19], la réponse est 3.

    Voici l'algorithme :

    Entrée : un tableau T de taille n (n > 0)
Sortie : le plus petit élément de T

1. mini ← T[0]
2. Pour i allant de 1 à n-1 :
3.     Si T[i] < mini :
4.         mini ← T[i]
5. Retourner mini

    Exemple tracé pas à pas pour T = [4, 9, 3, 19] :

    Étape i T[i] mini
    Départ - - 4
    i = 1 1 9 4 (9 > 4, on ne change pas)
    i = 2 2 3 3 (3 < 4, on met à jour)
    i = 3 3 19 3 (19 > 3, on ne change pas)

    L'algorithme renvoie 3. Correct !

    Pourquoi ça marche ? À chaque étape, mini contient le plus petit élément parmi ceux qu'on a déjà vus. À la fin, on a tout vu : mini est donc le minimum de tout le tableau.

    Terminaison

    Un algorithme doit toujours s'arrêter. Pour FIND-MIN, la boucle passe exactement une fois par chaque indice de 1 à n-1. Elle s'arrête nécessairement après n-1 étapes.

    On distingue :

    • Boucle bornée (for) : le nombre d'itérations est fixé à l'avance. Elle se termine toujours.
    • Boucle non bornée (while) : elle s'arrête quand une condition devient vraie. Il faut s'assurer que cette condition finira par être atteinte.

    Exercice 3 - Bornée ou non bornée ?

    Pour chaque boucle, dire si elle est bornée et si elle se termine toujours :

    1. for i in range(50):
    2. while n > 0: n -= 1 (avec n >= 0 au départ)
    3. while True: pass
    4. i = 10 puis while i > 0: i -= 3
    Correction
    1. Bornée, se termine toujours : exactement 50 itérations.
    2. Non bornée, se termine toujours : n décroît à chaque étape et atteint 0.
    3. Non bornée, ne se termine jamais : boucle infinie, aucune condition d'arrêt.
    4. Non bornée, se termine toujours : i vaut 10, 7, 4, 1, puis -2 < 0, la boucle s'arrête.

    Complexité : combien d'étapes faut-il ?

    Un algorithme peut être correct mais trop lent pour être utile. La complexité mesure le nombre d'étapes en fonction de la taille de l'entrée \(n\).

    Pour FIND-MIN, on parcourt le tableau une seule fois : le nombre d'étapes est proportionnel à \(n\). On dit que la complexité est linéaire.

    Les grandes familles de complexité, de la plus rapide à la plus lente :

    Complexité Nom Exemple
    \(\mathcal{O}(1)\) Constante Accéder à T[5]
    \(\mathcal{O}(n)\) Linéaire Chercher le minimum
    \(\mathcal{O}(n^2)\) Quadratique Tri par sélection
    \(\mathcal{O}(2^n)\) Exponentielle Certains problèmes de TSP

    Un algorithme en \(\mathcal{O}(n^2)\) sera bien plus lent qu'un algorithme en \(\mathcal{O}(n)\) dès que \(n\) devient grand. Pour \(n = 1000\), le premier fait environ 1 000 000 d'opérations, le second seulement 1 000.

    Exercice 4 - Tracer FIND-MIN

    Tracer pas à pas l'algorithme FIND-MIN pour le tableau [7, 2, 8, 1, 5]. Remplir le tableau d'évolution de mini à chaque étape.

    Correction
    Étape i T[i] mini
    Départ - - 7
    i = 1 1 2 2 (2 < 7)
    i = 2 2 8 2 (8 > 2)
    i = 3 3 1 1 (1 < 2)
    i = 4 4 5 1 (5 > 1)

    L'algorithme renvoie 1.

    Exercice 5 - Trouver le maximum

    Modifier l'algorithme FIND-MIN pour qu'il trouve le maximum au lieu du minimum. Écrire le pseudocode, puis le tracer pour [7, 2, 8, 1, 5].

    Correction 
    1. maxi ← T[0]
2. Pour i allant de 1 à n-1 :
3.     Si T[i] > maxi :
4.         maxi ← T[i]
5. Retourner maxi

    Trace pour [7, 2, 8, 1, 5] :

    Étape i T[i] maxi
    Départ - - 7
    i = 1 1 2 7 (2 < 7)
    i = 2 2 8 8 (8 > 7)
    i = 3 3 1 8 (1 < 8)
    i = 4 4 5 8 (5 < 8)

    L'algorithme renvoie 8.

    Exercice 6 - Identifier la complexité

    Pour chaque algorithme décrit, identifier sa complexité parmi : constante, linéaire, quadratique.

    1. Afficher le premier élément d'un tableau.
    2. Calculer la somme de tous les éléments d'un tableau de taille \(n\).
    3. Pour chaque élément d'un tableau, le comparer à tous les autres éléments.
    Correction
    1. Constante \(\mathcal{O}(1)\) : une seule opération, quelle que soit la taille du tableau.
    2. Linéaire \(\mathcal{O}(n)\) : on parcourt le tableau une fois.
    3. Quadratique \(\mathcal{O}(n^2)\) : pour chaque élément (×n), on parcourt tout le tableau (×n).
