## Afficher quelque chose

L'instruction `print` affiche un message à l'écran. Ce qui est entre guillemets est affiché tel quel.

!!! note "Pourquoi « print » ?"
    `print` veut dire **imprimer** en anglais. Aux débuts de l'informatique, il n'y avait pas d'écran : les résultats sortaient sur du **papier**, imprimés. Les écrans sont arrivés ensuite, mais le mot, lui, est resté.

```python
print("Bonjour")
print("Bienvenue en SNT")
```

Ce programme affiche deux lignes :

```text
Bonjour
Bienvenue en SNT
```

Attention à un point important : les **guillemets** changent tout. Compare :

```python
print("3 + 4")
print(3 + 4)
```

Ce programme affiche :

```text
3 + 4
7
```

La première ligne est **entre guillemets** : c'est du texte, recopié tel quel. La seconde est **sans guillemets** : Python la lit comme un calcul et affiche le résultat.

## Calculer

Sans guillemets, Python fait les calculs. Il connaît `+`, `-`, `*` (multiplier) et `/` (diviser).

```python
print(2026 - 2009)
print(6 * 7)
```

affiche :

```text
17
42
```

L'intérêt d'écrire un calcul plutôt que le résultat, c'est que Python ne se trompe jamais dans le calcul, et que si les nombres changent, le programme se met à jour tout seul.

## Lire, puis modifier

En programmation, on passe beaucoup de temps à **lire du code écrit par d'autres** et à le **modifier**, avant d'écrire le sien de zéro. C'est normal, et on s'y entraînera toute l'année.

Savoir lire, c'est aussi savoir repérer une erreur. Par exemple, ce programme ne fonctionne pas :

```python
print(Bonjour)
```

Il manque les guillemets : `Bonjour` sans guillemets n'est pas du texte pour Python, qui affiche une erreur. La version correcte est :

```python
print("Bonjour")
```

## Ce qu'on retient

- **Python** exécute vraiment les algorithmes qu'on écrit.
- `print(...)` affiche : entre **guillemets** c'est du texte recopié, **sans guillemets** un calcul est effectué.
- On progresse d'abord en **lisant et en modifiant** du code existant.
- Le cours est ici ; on s'entraîne sur **Capytale**.
